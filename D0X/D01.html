Ethan's Design Principles

Fundamental Values

  • Visibility of functions, instructions, etc.
  • Be consistent
  • Give Feedback: "response" from an object
  • Affordance: an attribute that allows people to know how to use it
  • Have familiar navigation

Complication Analysis

  • There is a limit on how simple an app can be
  • Use icons
  • Allow users to learn by doing
  • "Be as easy as email!"
  • Minimize choices, especially when response times are critical
  • Break complex tasks into smaller steps
  • Avoid overwhelming users
  • Be careful not to simplify into point of abstraction

Societal Factors

  • Individualism versus comminitarianism
  • Neutral versus emotional
  • Universalism versus particularism
  • Achievement versus ascription
  • Sequential versus synchronous time
  • Internal direction versus outer direction

Abstract Perspectives

  • A mental model is the user's representation
  • Design the conceptual model purposely
  • Be wary of a poor or erroneous model
  • Make sure the mental model and the conceptual model match
  • Metaphors need to have affrodances and constraints

User-Driven Design Principles

  • Provide discoverability
  • Provide feedback
  • Utilize a conceptual model
  • Include affordances, signifiers, and constraints
  • Ensure you have good mappings

Accessibility Guidelines

  • Use multiple cues
  • Clear instructions
  • For text, use text-to-speech
  • Use Alt-text for images

Mobile Experience Guidelines

  • Minimize clutter
  • Use a scanable interface
  • Keep buttons in comfortable positions
  • Allow for zooming
  • Have responsive feedback
  • Eliminate distractions
